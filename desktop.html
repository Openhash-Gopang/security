HOW IT WORKS

이렇게 작동합니다

각 하위 시스템에 에이전트를 한 줄 추가하면, K-Security가 자동으로 감시를 시작합니다.

1

에이전트 부착 — 단 한 줄

각 하위 시스템의 </body> 직전에 security-agent.js를 추가합니다. data-svc만 변경하면 22개 시스템 모두 동일하게 적용됩니다.

2

자가 진단 및 보고

에이전트는 30초마다 응답시간, 인증 상태, PDV 흐름을 자가 진단하고 K-Security에 6하원칙 보고서를 전송합니다. 페이지 언로드 시에도 sendBeacon으로 마지막 보고를 전송합니다.

3

AI 2단계 분석

DeepSeek V3가 보고서를 1차 스크리닝하고, 의심 징후가 있으면 Claude Opus가 심층 분석합니다. 22개 공격 패턴과 대조하여 S1~S3 심각도를 판정합니다.

4

이해관계자 알림 및 대응 지시

이상 감지 시 해당 시스템 이해관계자에게 gopang 앱으로 긴급 메시지를 전송합니다. S3(위험) 수준에서는 K-Police 자동 연동과 시스템 SUSPEND 지시가 발동됩니다.

<!-- 모든 하위 시스템 공통 — data-svc만 변경 -->
<script
  src="https://security.gopang.net/security-agent.js"
  data-svc="school"
  data-url="school.gopang.net"
></script>
FEATURES

핵심 기능

상시 감시

30초 간격으로 22개 시스템을 자동 점검합니다. 페이지 전환 시에도 백그라운드에서 계속 작동합니다.

인증 통합

subsystem-auth.js v1.1에 통합되어 있습니다. 기존 인증 한 줄에 보안 에이전트가 자동으로 포함됩니다.

OpenHash 특별 감시

해시체인 불일치, ILMV 검증 실패 1건만으로 즉시 S3 위험으로 판정합니다. 생태계 무결성의 최후 방어선입니다.

완전 공개 투명성

모든 보안 현황은 인증 없이 누구나 확인할 수 있습니다. 숨김으로써 보안을 달성하지 않습니다.

원격 지시 이행

K-Security가 각 시스템에 SUSPEND, SHOW_ALERT, RELOAD 등 7가지 지시를 원격으로 발동합니다.

K-Police 자동 연동

S3 위험 수준 이상에서는 K-Police에 자동 통보합니다. 사이버 범죄 대응을 위한 증거 패키지를 자동 생성합니다.

SEVERITY

심각도 판정 기준

이상 징후 점수(0.0~1.0)와 공격 유형에 따라 S1~S3로 판정하며, 각 단계별 대응이 자동으로 실행됩니다.

등급 이상 점수 상황 자동 대응
OK 0.0 ~ 0.4 정상 운영 없음
S1 0.4 ~ 0.6 응답 지연, PDV 대기열 증가, 단발성 오류 운영자 gopang 앱 알림
S2 0.6 ~ 0.85 연속 인증 실패, 프롬프트 인젝션 시도 긴급 알림 + 점검 간격 5초
S3 0.85 ~ DOUBLE_SPEND, 해시체인 단절, SYBIL_ATTACK 전체 알림 + K-Police + SUSPEND
